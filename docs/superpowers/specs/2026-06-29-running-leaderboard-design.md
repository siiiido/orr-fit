# [Design Spec] 피트니스 센터용 실시간 러닝 마일리지 대시보드

이 문서는 젬스톤피트니스 서면점 회원을 위한 실시간 러닝 마일리지 랭킹 웹사이트의 디자인 및 아키텍처 명세입니다.

---

## 1. 개요 및 요구사항 (Overview & Requirements)

*   **목적**: 피트니스 센터 회원의 달리기 기록을 취합하여 월간 누적 거리 순위와 실시간 챌린지 달성도를 시각적으로 매력적이게 보여줌으로써 운동 동기를 부여합니다.
*   **사용자 플로우**:
    1.  회원은 자신의 러닝 앱(나이키, 스트라바 등) 기록 화면을 캡처하여 센터 직원에게 SMS/메신저로 전송합니다.
    2.  센터 직원은 웹 대시보드에 내장된 관리자 모드(패스코드 인증)에 접속하여 회원의 이름, 거리, 시간 기록을 입력합니다.
    3.  입력 완료 즉시, 개인 모바일 폰으로 접속한 회원들과 센터 TV 모니터의 순위표가 실시간으로 갱신됩니다.
*   **스택**: React + TypeScript + Tailwind CSS + Supabase (무료 실시간 데이터베이스)
*   **폰트 및 메인 컬러**: Pretendard 폰트 / 주황색(Vibrant Orange) 메인 테마

---

## 2. 디자인 시스템 & 비주얼 테마 (Design System)

*   **테마**: 스포티 다크 테마 (Sporty Dark-First Design)
*   **주요 색상**:
    *   `primary` (오렌지): `#FF6B00` (활기차고 역동적인 주황색)
    *   `secondary` (골드/옐로우): `#FFC700` (우승자 왕관 및 포인트 강조용)
    *   `background` (배경): `#0B0C0E` ~ `#121318` (깊은 어두운 차콜 계열)
    *   `surface` (카드): `rgba(25, 27, 34, 0.7)` (반투명 글래스모피즘 효과, border: `rgba(255, 107, 0, 0.1)`)
*   **효과**:
    *   오렌지 그라데이션 글로우 효과 (`shadow-[0_0_15px_rgba(255,107,0,0.35)]`)
    *   순위 변동 및 신규 기록 등록 시 부드러운 트랜지션 애니메이션
*   **타이포그래피**:
    *   기본 서체: `Pretendard, -apple-system, BlinkMacSystemFont, system-ui, sans-serif`
    *   수치 서체: 거칠고 볼드한 느낌의 숫자 연출을 위해 Pretendard ExtraBold/Black 서체 사용

---

## 3. 기능 구조 및 컴포넌트 설계 (Features & Architecture)

단일 페이지 애플리리케이션(SPA) 내에서 탭(Tab) 또는 모달(Modal) 상태로 조회 모드와 관리자 모드를 전환합니다.

```
+-----------------------------------------------------------------------+
|  [Logo] ORR FIT (젬스톤피트니스 서면점)             [직원 전용 자물쇠]  |
+-----------------------------------------------------------------------+
|                                                                       |
|  +--------------------+  +--------------------+  +-----------------+  |
|  |  공동 목표 달성도  |  |    명예의 전당     |  | 실시간 러닝피드 |  |
|  |   (서클 게이지)    |  |   (1~3위 포디움)   |  | (최근 달성 로그)|  |
|  |                    |  |                    |  |                 |  |
|  |  [  71.5% 달성  ]  |  |   4위~10위 리스트  |  |  - 길동님 5km   |  |
|  |   목표: 2,026 km   |  |   (회원 검색창)    |  |  - 영희님 10km  |  |
|  +--------------------+  +--------------------+  +-----------------+  |
|                                                                       |
+-----------------------------------------------------------------------+
```

### 3.1 회원용 메인 대시보드
1.  **목표 달성 카드 (Goal Card)**:
    *   월간 누적 목표 거리(예: 2,026km) 대비 현재 회원들의 총합 러닝 거리 비율을 서클 게이지 및 프로그레스 바로 표현.
2.  **명예의 전당 포디움 (Top 3 Podium)**:
    *   1위(금/왕관), 2위(실버), 3위(브론즈) 회원을 시상대 형태로 디자인하여 시각적 강조.
3.  **순위 리스트 (Leaderboard List)**:
    *   전체 회원 순위 표. 닉네임 검색 필터 기능 내장.
4.  **최근 활동 피드 (Recent Feed)**:
    *   누가 몇 km를 달렸는지 실시간으로 흘러가는 피드. '박수 보내기' 클릭 시 카운트 증가(애니메이션 효과).

### 3.2 직원용 관리 모드 (Admin Panel)
*   **보안 게이트**: 우측 상단 자물쇠 클릭 -> 패스코드(예: `0000`) 입력 시 관리자 패널 활성화.
*   **기능**:
    *   **신규 회원 등록**: 이름(닉네임), 성별 추가.
    *   **러닝 기록 추가**: 회원명 검색 선택 -> 거리(km), 시간(시:분:초), 날짜(선택), 확인 메모 입력 -> 데이터베이스 저장.
    *   **기록 관리**: 잘못 입력된 회원 데이터나 러닝 로그 삭제 및 수정 기능.
    *   **공동 목표 설정**: 이번 달 센터 전체 목표 거리 설정 변경 가능.

## 4. 데이터베이스 스키마 (Supabase Schema)

Supabase PostgreSQL 데이터베이스에 다음 테이블들을 생성합니다.

### 4.1 `members` 테이블 (회원 정보)
*   `id`: `uuid` (Primary Key, default: `gen_random_uuid()`)
*   `name`: `text` (Unique, 회원 이름/닉네임)
*   `gender`: `text` (성별: 'M', 'F')
*   `created_at`: `timestamp with time zone` (default: `now()`)

### 4.2 `runs` 테이블 (러닝 기록)
*   `id`: `uuid` (Primary Key, default: `gen_random_uuid()`)
*   `member_id`: `uuid` (Foreign Key -> `members.id` ON DELETE CASCADE)
*   `distance`: `numeric` (달린 거리, km 단위, 예: 5.25)
*   `duration`: `integer` (달린 시간, 초 단위, 예: 1800)
*   `notes`: `text` (직원 확인 메모, 예: "스트라바 캡처 확인")
*   `created_at`: `timestamp with time zone` (default: `now()`)

### 4.3 `settings` 테이블 (센터 설정)
*   `key`: `text` (Primary Key, 예: 'monthly_target')
*   `value`: `jsonb` (설정 값, 예: `{"distance": 2026}`)

## 5. 실시간 동기화 아키텍처 (Realtime & State)

1.  **Supabase Realtime Channel**:
    *   React 앱 구동 시 `runs` 및 `members` 테이블의 `INSERT`, `UPDATE`, `DELETE` 이벤트를 구독(Subscribe)합니다.
    *   직원이 데이터를 추가하는 즉시 모든 접속 회원의 React State가 실시간 갱신되어 새로고침 없이 대시보드 순위가 업데이트됩니다.
2.  **자동 페이스(Pace) 계산**:
    *   페이스 = 시간(분) / 거리(km)
    *   예: 5km를 25분 00초에 달린 경우 -> 페이스 = 25.0 / 5 = 5.0 (5'00" / km)
    *   이를 초 단위 데이터로 가공하여 `MM'SS"` 포맷으로 뷰어에 렌더링합니다.

---

## 6. 보안 및 예외 처리 (Security & UX Guardrails)

*   **입력값 유효성 검사 (Form Validation)**:
    *   거리는 숫자만 입력 가능 (소수점 첫째/둘째 자리까지 권장).
    *   시간은 `분:초` 또는 `시:분:초` 포맷으로 입력받아 서버에는 초 단위(`integer`)로 환산하여 저장.
*   **패스코드 보호**:
    *   관리자 권한은 로컬 클라이언트 상태(React Context/State) 및 간단한 패스코드로 통제하여 비개발자 직원이 손쉽게 진입하되 일반 회원의 터치를 방지합니다.
