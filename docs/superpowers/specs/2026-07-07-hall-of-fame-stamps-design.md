# 명예의 전당 도장 기능 설계서 (Hall of Fame Stamp Board Design Spec)

- **작성일:** 2026-07-07
- **상태:** 승인 완료 (User Approved)

본 문서는 오르핏(orr-fit) 웹 서비스의 "HALL OF FAME" 보드에 "명예의 전당 도장 현황" 모달을 추가하고, 매월 1~6등에게 부여되는 도장 기록을 관리자가 직접 등록/수정/자동 계산하여 관리할 수 있는 테이블 스키마 및 UI 설계를 정의합니다.

---

## 1. 개요 및 요구사항

### 목표
- 회원들이 매월 1~6등을 달성할 때마다 도장을 1개씩 획득하며, 6개의 도장을 다 모았을 때 특별한 상품을 제공하기 위한 시스템을 마련합니다.
- 사용자에게 전체 멤버들의 도장 현황을 볼 수 있는 직관적인 모달을 제공합니다. (모바일 친화적)
- 관리자가 이 기록을 수동으로 입력/수정하거나, 월별 러닝 기록(`runs`) 데이터를 기반으로 1~6등 순위를 손쉽게 자동 계산 및 저장할 수 있도록 기능을 구축합니다.

### 핵심 요구사항
1. **명예의 전당 도장 현황 모달:**
   - 1~6등 등극 시 획득한 도장 이력 및 도장 개수(0~6개) 표시
   - 모바일 환경에서의 가독성을 보장하며, 호버 없이도 직관적으로 획득 년-월 및 기록 정보를 텍스트로 조회 가능해야 함
2. **관리자 기능:**
   - 특정 월의 1~6등 내역을 조회/수정/삭제/저장할 수 있는 어드민 패널 페이지 기능 추가
   - 이번 달 러닝 기록에서 1~6등 순위 및 거리를 자동 추출하여 입력 폼에 바인딩하는 "자동 계산" 기능 구현
3. **데이터베이스 관리:**
   - 월별 1~6등 내역을 안전하게 별도 저장할 수 있는 테이블 마이그레이션 (`monthly_rankings`)

---

## 2. 데이터베이스 스키마 설계

새로운 데이터베이스 테이블인 `monthly_rankings`를 추가하여 월별 순위 및 거리 기록을 보존합니다.

```sql
-- 월별 1~6등의 기록 및 도장 정보를 관리하는 테이블
CREATE TABLE monthly_rankings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  member_id UUID REFERENCES members(id) ON DELETE CASCADE NOT NULL,
  year_month VARCHAR(7) NOT NULL, -- 형식: 'YYYY-MM' (예: '2026-07')
  rank INTEGER NOT NULL CHECK (rank >= 1 AND rank <= 6),
  distance NUMERIC(6,2) NOT NULL CHECK (distance >= 0),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL,
  
  -- 한 달에 특정 등수(1~6등)는 오직 한 명만 존재할 수 있음
  CONSTRAINT unique_year_month_rank UNIQUE (year_month, rank),
  -- 한 회원은 한 달에 하나의 순위 기록(도장)만 가질 수 있음
  CONSTRAINT unique_member_year_month UNIQUE (member_id, year_month)
);

-- 실시간 동기화를 위해 realtime 게시판에 추가
ALTER PUBLICATION supabase_realtime ADD TABLE monthly_rankings;
```

## 3. UI 및 컴포넌트 설계

### 3.1. 메인 화면 진입로 (`Leaderboard.tsx`)
- **버튼 추가:**
  - `Leaderboard` 컴포넌트의 "Hall of Fame" 헤더 영역 근처에 `🏆 명예의 전당 도장판 보기` 버튼 배치
  - 오렌지 및 골드 톤의 그라데이션 및 활력 있는 그림자 효과(Shadow/Pulse)를 적용하여 클릭을 유도
  - 클릭 시 `StampsModal`을 활성화

### 3.2. 명예의 전당 도장 현황 모달 (`StampsModal.tsx` - 신설)
- **모바일 대응 레이아웃:**
  - 회원 이름/닉네임 검색용 인풋 필드
  - 회원별 카드 그리드/리스트
  - **멤버 카드 상세 정보:**
    - 이름, 닉네임, 성별, 총 획득 도장 개수 (`X / 6`)
    - 6개의 도장판 슬롯 (원형 모양)
      - **찍힌 도장 (Active):** 주황색/골드 원형 배경에 `7월`, `8월`과 같이 획득한 월 이름 텍스트 표기
      - **빈 도장 (Inactive):** 회색 점선 테두리의 빈 원형
      - **카드 하단 요약 텍스트:** `기록: 7월 2등(55.2km), 8월 5등(42.1km)` 형식으로, 어떤 월에 몇 등으로 몇 km를 기록했는지 모바일에서도 터치나 호버 없이 바로 텍스트로 보임
      - **올클리어 상태:** 도장을 6개 모두 모으면 골드 빛깔의 테두리 글로우 효과 및 `🎉 상품 대상자` 배지 활성화

### 3.3. 관리자 패널 (`AdminPanel.tsx`)
- **명예의 전당 관리 섹션 추가:**
  - 월 선택창 (예: `2026-07`)
  - **이번 달 기록 불러오기 (자동 계산 버튼):**
    - 클릭 시, 지정한 년-월에 축적된 `runs` 기록을 분석하여 멤버별 합산 거리를 내림차순 정렬하고 1~6위 목록을 자동으로 추출
    - 추출된 1~6위 목록 데이터를 입력 폼 필드에 자동으로 바인딩함
  - **1~6등 수동 수정 폼:**
    - 각 등수별 멤버 매핑 드롭다운(전체 멤버 리스트) 및 최종 달성 거리(km) 수동 입력창 제공
  - **저장 및 삭제 버튼:**
    - 해당 월의 순위 기록을 한 번에 데이터베이스에 Upsert(혹은 일괄 Delete 후 Insert) 하여 저장

## 4. 검증 계획

### 4.1. 수동 기능 테스트 시나리오
1. **관리자 기능 동작 검증:**
   - 관리자 패널에서 `2026-07` 년월을 설정하고 `이번 달 기록 불러오기` 클릭 시, 7월 러닝 데이터를 바탕으로 1~6등이 잘 매칭되는지 확인
   - 임의로 멤버와 거리 기록을 수동 수정한 뒤 `저장`했을 때, DB `monthly_rankings` 테이블에 올바르게 기록되는지 확인
2. **도장판 모달 뷰 테스트:**
   - 도장 획득이 없는 멤버, 1~5개 획득한 멤버, 6개 이상 획득한 멤버별로 도장 카드가 다르게 렌더링되는지 확인 (올클리어 6개 시 골드 효과 활성화)
   - 모바일 디바이스 크기(가로폭 360px~450px)에서 깨짐 현상 없이 한 행에 6개 도장 슬롯과 하단 텍스트가 조화롭게 배치되는지 검증
3. **실시간 동기화:**
   - 관리자가 데이터를 저장하면 화면을 새로고침하지 않아도 도장판 모달의 도장 획득 현황이 실시간으로 갱신되는지 확인
