# Database Backup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create a scheduled GitHub Actions workflow that backs up the Supabase database weekly and stores it as an artifact.

**Architecture:** A single GitHub Actions workflow (`.github/workflows/db-backup.yml`) utilizing `pg_dump` on `ubuntu-latest` and `actions/upload-artifact@v4`.

**Tech Stack:** GitHub Actions, PostgreSQL (`pg_dump`).

## Global Constraints

- The connection string will be provided via GitHub Repository Secrets named `SUPABASE_DB_URL`.
- The backup artifact retention is 90 days.
- The cron schedule is `0 0 * * 0` (Every Sunday at 00:00 UTC).

---

### Task 1: Create Backup Workflow

**Files:**
- Create: `.github/workflows/db-backup.yml`

**Interfaces:**
- Consumes: GitHub Secrets (`SUPABASE_DB_URL`).
- Produces: GitHub Artifact (`orr-fit-db-backup.sql`).

- [ ] **Step 1: Create the workflow file**

Create the `.github/workflows/db-backup.yml` file with the following content:

```yaml
name: Database Backup

on:
  schedule:
    - cron: '0 0 * * 0' # Every Sunday at 00:00 UTC
  workflow_dispatch: # Allow manual trigger for testing

jobs:
  backup:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Install PostgreSQL Client
        run: sudo apt-get update && sudo apt-get install -y postgresql-client

      - name: Dump Database
        env:
          SUPABASE_DB_URL: ${{ secrets.SUPABASE_DB_URL }}
        run: |
          BACKUP_DATE=$(date +'%Y-%m-%d')
          pg_dump --clean --if-exists --no-owner --no-privileges --quote-all-identifiers -d "$SUPABASE_DB_URL" -f "orr-fit-db-backup-$BACKUP_DATE.sql"

      - name: Upload Backup Artifact
        uses: actions/upload-artifact@v4
        with:
          name: orr-fit-db-backup
          path: orr-fit-db-backup-*.sql
          retention-days: 90
```

- [ ] **Step 2: Commit the workflow**

```bash
git add .github/workflows/db-backup.yml
git commit -m "ci: add scheduled database backup workflow"
```
